# Founder OS Blueprint

Blueprint initial pour un Founder OS oriente micro-agence de developpement web assistee par IA.

## Structure

- `docs/business-brief.md` : cadrage business du projet
- `docs/architecture.md` : architecture cible et schema Mermaid
- `docs/agent-roles.md` : fiches agents et regles d'intervention
- `docs/permissions-policy.md` : politique de permissions
- `docs/provider-choice.md` : choix provisoires local/cloud
- `docs/run-journal.md` : journal day 1
- `evidence/screenshots/` : captures a produire au fil des runs
- `evidence/runs/` : traces et comptes rendus d'execution
- `exports/` : exports utiles au projet
- `vault/` : memoire Obsidian / base de connaissance locale

## Projet retenu

Le cas par defaut retenu ici est `Web Studio OS` : une micro-agence qui utilise un systeme multi-agents pour accelerer la prospection, le cadrage, la production web, le suivi commercial et l'operationnel.

## Etat

Jour 1 : fondations documentaires uniquement. Pas de systeme automatise deploye a ce stade.

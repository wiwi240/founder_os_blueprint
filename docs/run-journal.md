# Run Journal

## Day 1 - 2026-07-16

### Pourquoi ce projet

J'ai retenu `Web Studio OS` car le niveau de douleur est concret : une micro-agence subit beaucoup de micro-taches repetitives qui cassent le focus, ralentissent la vente et degradent la qualite d'execution.

### Agents les plus importants

1. Mail / Sales : car la vitesse de reponse et la clarte commerciale ont un impact direct sur la conversion.
2. Code / Produit : car un mauvais cadrage detruit ensuite la marge et la livraison.
3. Prospection : car sans pipeline qualifie, le systeme est inutile.

### Actions sensibles

- envoi d'emails reels
- ecriture dans la memoire permanente
- import/export de donnees prospects ou clients
- generation de documents commerciaux engageants
- usage d'outils ou d'API payantes

### Questions pour demain

- Quel format exact doit prendre le vault : fiches Markdown simples ou structure plus normalisee ?
- Faut-il un CRM minimal ou seulement des notes + exports CSV ?
- Quelle granularite de validation humaine est acceptable sans ralentir tout le systeme ?
- Quelles preuves minimales doit-on conserver pour auditer une action agentique ?

### Decision de scope

Je limite le jour 1 au blueprint. Construire de vraies integrations maintenant serait premature. Le bon choix MVP est de clarifier les roles, les permissions, les flux et les preuves attendues avant toute automatisation.
